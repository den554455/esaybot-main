# Шпаргалка: изменения и деплой

## Внести изменения и задеплоить

```bash
# 1. Перейти в папку проекта
cd ~/my-app

# 2. Посмотреть что изменилось
git status

# 3. Добавить все изменённые файлы
git add .

# 4. Сохранить изменения с описанием
git commit -m "что именно сделали"

# 5. Отправить на GitHub (автоматически запустит деплой)
git push
```

После `git push` GitHub Actions сам соберёт проект и задеплоит на Yandex Cloud.  
Следить за процессом: вкладка **Actions** в репозитории на GitHub.

---

## Примеры описаний для коммитов

```bash
git commit -m "fix: исправлена ошибка регистрации"
git commit -m "add: добавлена страница профиля"
git commit -m "update: обновлён дизайн главной страницы"
git commit -m "fix: исправлен CORS на бэкенде"
```

---

## Посмотреть историю изменений

```bash
git log --oneline
```

---

## Если что-то пошло не так — откатиться

```bash
# Посмотреть историю
git log --oneline

# Откатить последний коммит (изменения останутся в файлах)
git revert HEAD
git push
```

---

## Типичный сценарий — исправили баг

```bash
cd ~/my-app
# отредактировали нужный файл...
git add .
git commit -m "fix: исправлена ошибка CORS"
git push
# через 2-3 минуты изменения на сайте
```

---

## Полезные команды

| Команда | Что делает |
|---|---|
| `git status` | показывает что изменилось |
| `git add .` | добавляет все изменения |
| `git add файл` | добавляет конкретный файл |
| `git commit -m "..."` | сохраняет изменения |
| `git push` | отправляет на GitHub |
| `git log --oneline` | история коммитов |
| `git diff` | что именно изменилось в файлах |
