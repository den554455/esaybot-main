# Список исправлений и улучшений

## 🐛 Исправленные ошибки

### api.js
- Добавлен таймаут (8 сек) для всех fetch-запросов через `fetchWithTimeout`
- `bookAppointment` и `cancelAppointment` теперь обёрнуты в try/catch
- API ключ вынесен в env-переменную (`REACT_APP_API_KEY`)

### MastersPage.js
- Фильтрация по цене теперь работает (раньше была закомментирована)
- Исправлено: прямой переход на `/masters` без выбора услуги перенаправляет на главную
- Фильтр срабатывает при каждом изменении текста, а не только по кнопке
- Добавлен fallback для пустого списка с кнопкой сброса

### CalendarPage.js
- Прямой переход на `/calendar` без master/service перенаправляет на главную
- Нельзя выбрать прошедшую дату
- Нельзя листать недели в прошлое
- Отдельный loading-стейт для кнопки "Записаться" (предотвращает двойную отправку)
- Слоты загружаются сразу при открытии страницы

### CabinetPage.js
- `window.location.href = '/'` заменён на `navigate('/')`
- Безопасный доступ к вложенным полям (`appt.service?.name`)
- Безопасный вызов `.toLocaleString()` через `Number()`
- Отдельный стейт `cancelling` (блокирует повторную отмену)

### MasterPanelPage.js
- Ошибка входа показывается под полем, не через `alert()`
- Enter теперь отправляет форму входа
- Добавлена кнопка выхода из панели
- Заглушка `handleConfirm` обновляет локальный стейт

## ✨ Улучшения

- **MastersPage**: баннер выбранной услуги, счётчик результатов, стрелка на карточке, доступность клавиатуры
- **CalendarPage**: бейдж услуги в заголовке, текст "нет слотов"
- **CabinetPage**: toast-уведомление после записи, иконка пустого состояния
- **MasterPanelPage**: иконка, подзаголовок, кнопка выхода, badge подтверждённых записей

## ⚠️ Важно сделать перед продакшеном

1. Создать `.env` файл из `.env.example` и задать `REACT_APP_API_KEY`
2. Убрать `DEV_PASSWORD` из MasterPanelPage — реализовать авторизацию через API
3. Реализовать настоящую авторизацию пользователей (не хардкод `DEMO_USER_ID`)
